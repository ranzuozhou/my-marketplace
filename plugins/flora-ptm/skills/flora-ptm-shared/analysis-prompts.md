# Flora PTM 按报告类型的分析 Prompt 变体

本文档包含 digest Phase 2 逐篇分析时，根据报告类型选择的 prompt 变体。默认通用版见 `→ prompt-templates.md` §1。

---

## 选择逻辑

在 digest Phase 1 导入时，根据文件特征自动推荐变体，用户可覆盖：

| 特征 | 推荐变体 | 判断依据 |
|------|----------|----------|
| `.pdf` + 来源为学术数据库 URL | 学术论文版 | URL 含 arxiv, doi, scholar, ieee, acm |
| `.pdf` + 文件名含公司/机构名 | 行业报告版 | 文件名模式匹配 |
| URL 来源为博客平台 | 博客/观点文章版 | URL 含 medium, substack, blog, zhihu, mp.weixin |
| `.md` / `.txt` | 通用版 | 纯文本难以判断类型 |
| 无法判断 | 通用版 | 让用户在人工判断点覆盖 |

**人工覆盖**: 在 digest Phase 2 人工判断点，如果用户发现某篇分析不够深入，可指定换用其他变体重新分析。

## 学术论文版

适用于: 学术论文、预印本、会议论文

```
请对这篇学术论文进行深度分析，生成结构化的精简短文：

## 研究问题与动机
这篇论文要解决什么问题？为什么这个问题重要？

## 主题与关键词
列出 3-5 个核心关键词（含领域术语）

## 方法论评估
- 研究设计类型（实验/观察/理论/综述）
- 数据集和样本规模
- 实验设计是否充分？有无对照组？
- 统计方法是否恰当？

## 核心发现与论证逻辑
- 主要实验结果及量化指标
- 论证逻辑链：假设 → 实验 → 结论
- 结果的统计显著性和实际意义

## 创新贡献
- 相对于先前工作的增量贡献
- 提出的新概念、方法或框架
- 对领域的潜在影响

## 局限性与未来方向
- 作者自述的局限性
- 审稿人可能提出的质疑
- 未解决的问题和建议的后续研究

## 引用网络要点
- 最关键的 3-5 篇被引文献及其与本文的关系
- 本文在该领域引用图谱中的位置

## 一句话核心价值
用一句话概括这篇论文最值得记住的核心贡献
```

## 行业报告版

适用于: 咨询公司报告、行业白皮书、市场研究、政策报告

```
请对这份行业报告进行深度分析，生成结构化的精简短文：

## 报告背景与目的
- 发布机构及其立场/利益关系
- 报告的目标受众和核心目的

## 主题与关键词
列出 3-5 个核心关键词

## 核心数据与结论
- 关键数据点和统计数字
- 主要结论和预测
- 数据来源和采集方法

## 数据可靠性评估
- 样本量和覆盖范围是否充分？
- 数据来源是否权威和独立？
- 是否存在选择性使用数据的倾向？

## 市场假设验证
- 报告基于哪些核心假设？
- 这些假设是否经过论证？
- 如果假设不成立，结论如何变化？

## 利益相关方分析
- 报告结论对不同利益方的影响
- 发布方可能的偏见或商业动机

## 可操作洞察
- 对决策者最有价值的 3 个行动建议
- 建议的时效性和适用条件

## 一句话核心价值
用一句话概括这份报告最值得关注的核心发现
```

## 博客/观点文章版

适用于: 技术博客、专栏文章、个人分析、社交媒体长文

```
请对这篇观点文章进行深度分析，生成结构化的精简短文：

## 作者背景与立场
- 作者的专业背景和可能的立场偏好
- 文章的写作目的和目标受众

## 主题与关键词
列出 3-5 个核心关键词

## 核心观点与论证
- 文章的中心论点
- 使用了哪些论证方式（数据、案例、类比、权威引用）
- 论证链是否完整和有说服力

## 论证质量评估
- 论据是否充分支撑结论？
- 是否存在逻辑跳跃或因果混淆？
- 引用的数据/案例是否经过选择性呈现？

## 偏见识别
- 确认偏见：是否只引用支持观点的证据？
- 幸存者偏见：是否忽略了失败案例？
- 时效偏见：观点是否受特定时间窗口影响？
- 文化偏见：观点是否仅适用于特定语境？

## 独特价值
- 文章提供了哪些其他来源难以获取的视角？
- 作者的实践经验带来了哪些独特洞察？

## 适用边界
- 文章观点在什么条件下成立？
- 推广到其他场景时需要注意什么？

## 一句话核心价值
用一句话概括这篇文章最值得记住的核心洞察
```

## 自定义变体

如果用户的材料不属于以上三类（如：专利文档、法律文件、教科书章节等），在 digest Phase 2 人工判断点，用户可以口述希望重点分析的维度，由 Claude 动态生成定制化的分析 prompt。

定制 prompt 应保留以下固定部分：
- **主题与关键词**（3-5 个）
- **核心观点与论证逻辑**
- **一句话核心价值**

其余维度根据用户需求替换。
